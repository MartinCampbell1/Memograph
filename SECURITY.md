# Security Policy

## Supported versions

| Version | Supported |
| --- | --- |
| `main` | Yes |
| Preview tags (`v0.x`) | Best effort |
| Older preview builds | No |

## Reporting a vulnerability

Use GitHub's private vulnerability reporting for this repository through the Security tab.

When reporting, include:

- affected version or commit
- exact reproduction steps
- expected impact
- whether the issue involves local data exposure, provider credentials, permissions, or release artifacts

## Response targets

- Initial acknowledgement: within 5 business days
- Triage decision: within 10 business days
- Fix ETA: depends on severity and reproducibility

## Disclosure expectations

- Do not open public issues for unpatched vulnerabilities.
- Keep reports private until a fix or mitigation is available.
- Privacy-sensitive bugs involving screenshots, OCR text, microphone capture, or exported notes are treated as high priority.
